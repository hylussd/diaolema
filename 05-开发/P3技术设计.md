# 钓了吗 P3 商城 技术方案设计

## 1. 概述

- **版本：** P3
- **背景：** P2 已完成社区铺垫（众包数据/配方/评分）；P3 在此基础上构建商城基础，提供装备展示与交易能力
- **设计原则：** 复用现有代码风格（FastAPI + SQLAlchemy + SQLite）；商城数据与现有标点/打卡数据正交；微信支付分阶段接入（先做订单流程，支付为 mock）

---

## 2. 数据模型设计

### 2.1 ER 关系（文字版）

```
users (1) ──── (N) orders
users (1) ──── (N) carts ──── (N) cart_items
product_categories (1) ──── (N) products
orders (1) ──── (N) order_items
```

### 2.2 新增表：product_categories（商品分类）

> **说明：** 商城商品的一级分类（饵料/鱼线/鱼钩/工具/服装等）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 分类 ID |
| name | TEXT | NOT NULL, UNIQUE | 分类名称 |
| icon | TEXT | | emoji 或图标名称 |
| sort_order | INTEGER | DEFAULT 0 | 排序权重（越大越靠前） |
| is_active | INTEGER | DEFAULT 1 | 是否启用（0=禁用） |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |

```sql
CREATE TABLE product_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    icon TEXT,
    sort_order INTEGER DEFAULT 0,
    is_active INTEGER DEFAULT 1,
    created_at DATETIME NOT NULL DEFAULT (datetime('now'))
);
```

**初始数据（seed）：**
```
INSERT INTO product_categories (name, icon, sort_order) VALUES

  ('钓竿钓椅', '🎣', 7),
  ('配件工具', '🧰', 6),
```

### 2.3 新增表：products（商品表）

> **说明：** 商城商品，支持规格（SKU）、库存、价格。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 商品 ID |
| category_id | INTEGER | FK → product_categories.id | 所属分类 |
| name | TEXT | NOT NULL | 商品名称 |
| description | TEXT | | 商品详情描述 |
| price | INTEGER | NOT NULL | 单价（分），避免浮点精度问题 |
| stock | INTEGER | DEFAULT 0 | 库存数量 |
| images | TEXT | | JSON 数组，图片 URL 列表 `["url1","url2"]` |
| specs | TEXT | | JSON 数组，规格选项 `[{name:"颜色",options:["红","蓝"]}]` |
| is_active | INTEGER | DEFAULT 1 | 是否上架（0=下架） |
| is_featured | INTEGER | DEFAULT 0 | 是否精选推荐 |
| sales_count | INTEGER | DEFAULT 0 | 销量计数（下单后+1） |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW | 更新时间 |

> **说明：** `price` 存分为整数，避免 JS 浮点问题；前端显示时 `/ 100` 转为元。

```sql
CREATE TABLE products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    category_id INTEGER NOT NULL REFERENCES product_categories(id),
    name TEXT NOT NULL,
    description TEXT,
    price INTEGER NOT NULL,
    stock INTEGER DEFAULT 0,
    images TEXT,
    specs TEXT,
    is_active INTEGER DEFAULT 1,
    is_featured INTEGER DEFAULT 0,
    sales_count INTEGER DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    updated_at DATETIME NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_products_active ON products(is_active);
```

### 2.4 新增表： carts（购物车 — 用户级别单购物车）

> **说明：** 每个用户只有一个购物车，购物车内每个商品一条 cart_items 记录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 购物车 ID |
| user_id | INTEGER | FK → users.id, UNIQUE | 所属用户，每人唯一购物车 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW | 更新时间 |

```sql
CREATE TABLE carts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL UNIQUE REFERENCES users(id),
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    updated_at DATETIME NOT NULL DEFAULT (datetime('now'))
);
```

### 2.5 新增表：cart_items（购物车项）

> **说明：** 购物车中每个 SKU（商品+规格组合）一条记录，支持数量增减。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 记录 ID |
| cart_id | INTEGER | FK → carts.id | 所属购物车 |
| product_id | INTEGER | FK → products.id | 商品 ID |
| quantity | INTEGER | NOT NULL, DEFAULT 1 | 购买数量 |
| specs | TEXT | | JSON，选中规格 `{size:"L", color:"红"}` |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW | 更新时间 |

> **约束：** UNIQUE(cart_id, product_id, specs)，同商品同规格只能有一条购物车记录，数量在 quantity 里累加。

```sql
CREATE TABLE cart_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cart_id INTEGER NOT NULL REFERENCES carts(id),
    product_id INTEGER NOT NULL REFERENCES products(id),
    quantity INTEGER NOT NULL DEFAULT 1,
    specs TEXT,
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    updated_at DATETIME NOT NULL DEFAULT (datetime('now')),
    UNIQUE(cart_id, product_id, specs)
);
```

### 2.6 新增表：orders（订单表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 订单 ID |
| order_no | TEXT | NOT NULL, UNIQUE | 订单号（前端展示用） |
| user_id | INTEGER | FK → users.id | 下单用户 |
| total_amount | INTEGER | NOT NULL | 订单总价（分） |
| status | TEXT | NOT NULL, DEFAULT 'pending' | 订单状态 |
| address_name | TEXT | NOT NULL | 收货人姓名 |
| address_phone | TEXT | NOT NULL | 收货人电话 |
| address_detail | TEXT | NOT NULL | 收货详细地址 |
| remark | TEXT | | 订单备注 |
| pay_status | TEXT | NOT NULL, DEFAULT 'unpaid' | 支付状态 |
| pay_time | DATETIME | | 支付时间 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT NOW | 更新时间 |

**订单状态（status）：** `pending` / `paid` / `shipped` / `delivered` / `cancelled`
**支付状态（pay_status）：** `unpaid` / `paid` / `refunded`

```sql
CREATE TABLE orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_no TEXT NOT NULL UNIQUE,
    user_id INTEGER NOT NULL REFERENCES users(id),
    total_amount INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending',
    address_name TEXT NOT NULL,
    address_phone TEXT NOT NULL,
    address_detail TEXT NOT NULL,
    remark TEXT,
    pay_status TEXT NOT NULL DEFAULT 'unpaid',
    pay_time DATETIME,
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    updated_at DATETIME NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_order_no ON orders(order_no);
```

### 2.7 新增表：order_items（订单项）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 记录 ID |
| order_id | INTEGER | FK → orders.id | 所属订单 |
| product_id | INTEGER | FK → products.id | 商品 ID |
| product_name | TEXT | NOT NULL | 下单时快照：商品名称 |
| product_price | INTEGER | NOT NULL | 下单时快照：单价（分） |
| quantity | INTEGER | NOT NULL | 购买数量 |
| specs | TEXT | | JSON，选中规格 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |

```sql
CREATE TABLE order_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL REFERENCES orders(id),
    product_id INTEGER NOT NULL REFERENCES products(id),
    product_name TEXT NOT NULL,
    product_price INTEGER NOT NULL,
    quantity INTEGER NOT NULL,
    specs TEXT,
    created_at DATETIME NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX idx_order_items_order_id ON order_items(order_id);
```

### 3.4 购物车接口

#### 3.4.1 获取购物车

```
GET /v1/shop/cart
```

**响应 200：**
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "items": [
      {
        "id": 1,
        "product_id": 1,
        "name": "野战蓝鲫 50g",
        "price": 1500,
        "images": ["https://..."],
        "specs": {"规格": "50g"},
        "quantity": 2,
        "subtotal": 3000
      }
    ],
    "total_amount": 3000,
    "total_count": 2
  }
}
```

#### 3.4.2 添加商品到购物车

```
POST /v1/shop/cart
```

**请求体：**
```json
{
  "product_id": 1,
  "quantity": 2,
  "specs": {"规格": "50g"}
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| product_id | int | ✅ | 商品 ID |
| quantity | int | ✅ | 数量（正数=加，负数=减） |
| specs | object | ❌ | 选中规格 `{规格: "50g"}` |

> **逻辑：** 如果 cart 中已存在（product_id + specs 相同），则 quantity 累加；否则新建。

**响应 200：**
```json
{
  "code": 0,
  "data": { "cart_id": 1, "item_id": 1 },
  "msg": "已加入购物车"
}
```

**响应 422（库存不足）：**
```json
{ "code": 422, "data": null, "msg": "库存不足，当前库存 5 件" }
```

#### 3.4.3 更新购物车商品数量

```

```

**请求体：**

> **逻辑：** quantity 覆盖更新；quantity=0 等同于删除。

**响应 200：** `{ "code": 0, "data": null, "msg": "更新成功" }`

#### 3.4.4 删除购物车商品

**响应 200：** `{ "code": 0, "data": null, "msg": "购物车已清空" }`

### 3.5 订单接口

#### 3.5.1 创建订单

```
POST /v1/shop/orders
```

**请求体：**
```json
{
  "address_name": "张三",
  "address_phone": "13800138000",
  "address_detail": "浙江省杭州市西湖区xxx路",
  "remark": "请发圆通",
  "pay_status": "unpaid"
}
```

> **说明：** 订单商品从当前用户购物车中全量结算（下单后清空购物车）。`pay_status` 传 `paid` 时为模拟支付成功（mock）。

**逻辑：**
1. 取出用户购物车所有 items
2. 校验每个商品库存是否充足（防止并发超卖：锁定 stock 更新）
3. 扣减每个商品库存（`stock -= quantity`）
4. 累加各商品小计得到 `total_amount`
5. 生成唯一 `order_no`（格式：`DLM{timestamp}{random4}`）
6. 创建 order + order_items，清空购物车

**响应 201：**
```json
{
  "code": 0,
  "data": {
    "order_id": 1,
    "order_no": "DLM17448768000001234",
    "total_amount": 3500,
    "status": "pending",
    "pay_status": "unpaid"
  },
  "msg": "订单创建成功"
}
```

**响应 422（库存不足）：**
```json
{ "code": 422, "data": null, "msg": "商品「野战蓝鲫」库存不足，当前库存 3 件" }
```

## 4. 前端页面设计

### 4.1 新增页面清单

| 页面 | 文件路径 | 说明 |
|------|----------|------|
| 商城首页 | `pages/shop/index` | 分类横滑 + 精选商品 + 新品列表 |

### 4.6 订单列表页（shop/orders）
