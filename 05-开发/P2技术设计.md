# 钓了吗 P2 社区铺垫 技术方案设计

## 1. 概述

- **版本：** P2
- **背景：** P1 已完成地图+标点CRUD+打卡+天气+AI推荐；P2 在此基础上建立水文数据众包机制，为后续 AI 推荐积累真实数据
- **设计原则：** 复用现有代码风格（FastAPI + SQLAlchemy + SQLite）；不做数据库迁移；不引入 ML 或 AI 内容；防垃圾机制内聚在后端服务层

---

## 2. 数据模型设计

### 2.1 ER 关系（文字版）

```
users (1) ──── (N) spot_checkins ──── (N) crowd_reports
users (1) ──── (N) recipes
users (1) ──── (N) spot_ratings ──── (N) fishing_spots
```

### 2.2 新增表：crowd_reports（水文上报表）

> **说明：** 用户在打卡后上报标点的水文数据（水温/溶氧量/鱼种）。上报必须关联最近一次打卡记录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 记录 ID |
| user_id | INTEGER | FK → users.id, NOT NULL, INDEX | 上报用户 |
| spot_id | INTEGER | FK → fishing_spots.id, NOT NULL, INDEX | 关联标点 |
| checkin_id | INTEGER | FK → spot_checkins.id, NOT NULL | 关联打卡（最近一次） |
| water_temp | REAL | NOT NULL | 水温（°C），范围 -5~40 |
| dissolved_oxygen | REAL | NOT NULL | 溶氧量（mg/L），范围 0~20 |
| fish_species | TEXT | NOT NULL | 鱼种（来自标准鱼种列表） |
| report_time | DATETIME | NOT NULL, DEFAULT NOW | 上报时间 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |

> **索引：** (user_id, spot_id, date(report_time)) 用于防垃圾频率限制查询

```sql
CREATE TABLE crowd_reports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    spot_id INTEGER NOT NULL,
    checkin_id INTEGER NOT NULL,
    water_temp REAL NOT NULL,
    dissolved_oxygen REAL NOT NULL,
    fish_species TEXT NOT NULL,
    report_time DATETIME NOT NULL DEFAULT (datetime('now')),
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (spot_id) REFERENCES fishing_spots(id),
    FOREIGN KEY (checkin_id) REFERENCES spot_checkins(id)
);
CREATE INDEX idx_reports_user_spot_date ON crowd_reports(user_id, spot_id, date(report_time));
CREATE INDEX idx_reports_spot_id ON crowd_reports(spot_id);
```

### 2.3 新增表：recipes（个人配方表）

> **说明：** 用户保存的气压+水温+鱼种+钓场类型的组合条件，可设为公开（匿名分享）或私有。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 配方 ID |
| user_id | INTEGER | FK → users.id, NOT NULL, INDEX | 创建用户 |
| name | TEXT | NOT NULL | 配方名称（用户自定义，如"春季鲫鱼配方"） |
| pressure_min | INTEGER | NOT NULL | 气压最小值（hPa），如 1000 |
| pressure_max | INTEGER | NOT NULL | 气压最大值（hPa），如 1020 |
| water_temp_min | REAL | NOT NULL | 水温最小值（°C），如 15.0 |
| water_temp_max | REAL | NOT NULL | 水温最大值（°C），如 25.0 |
| fish_species | TEXT | NOT NULL | 目标鱼种 |
| spot_type | TEXT | | 钓场类型：河流/湖泊/水库/池塘/其他，可空 |
| is_public | INTEGER | DEFAULT 0 | 是否公开（0=私有，1=公开） |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

```sql
CREATE TABLE recipes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    pressure_min INTEGER NOT NULL,
    pressure_max INTEGER NOT NULL,
    water_temp_min REAL NOT NULL,
    water_temp_max REAL NOT NULL,
    fish_species TEXT NOT NULL,
    spot_type TEXT,
    is_public INTEGER DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    updated_at DATETIME NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
CREATE INDEX idx_recipes_user_id ON recipes(user_id);
CREATE INDEX idx_recipes_public ON recipes(is_public) WHERE is_public = 1;
```

### 2.4 新增表：spot_ratings（钓点评分/点赞表）

> **说明：** 用户对公开标点进行评分（1-5星）和点赞（每人每标点各1次）。评分可覆盖更新，点赞可切换。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 记录 ID |
| spot_id | INTEGER | FK → fishing_spots.id, NOT NULL | 关联标点 |
| user_id | INTEGER | FK → users.id, NOT NULL | 评分用户 |
| rating | INTEGER | CHECK(rating BETWEEN 1 AND 5) | 评分（1-5星），可NULL（仅点赞时） |
| liked | INTEGER | DEFAULT 1 | 是否点赞（1=点赞，0=取消），默认点赞 |
| updated_at | DATETIME | NOT NULL | 最后更新时间 |
| created_at | DATETIME | NOT NULL, DEFAULT NOW | 创建时间 |

> **约束：** UNIQUE(spot_id, user_id)，每人每标点仅一条记录

```sql
CREATE TABLE spot_ratings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    spot_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    rating INTEGER CHECK(rating BETWEEN 1 AND 5),
    liked INTEGER DEFAULT 1,
    updated_at DATETIME NOT NULL DEFAULT (datetime('now')),
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (spot_id) REFERENCES fishing_spots(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE(spot_id, user_id)
);
CREATE INDEX idx_ratings_spot_id ON spot_ratings(spot_id);
```

### 2.5 扩展表：spot_checkins（打卡记录扩展）

> **说明：** P1 已实现打卡时间+鱼种+重量+天气；P2 扩展增加钓法、是否公开、水文关联字段。

| 新增字段 | 类型 | 约束 | 说明 |
|----------|------|------|------|
| fishing_method | TEXT | | 钓法：台钓/路亚/传统钓/飞蝇钓/海竿/手竿，可空 |
| is_public | INTEGER | DEFAULT 0 | 是否公开（公开则进入社区数据池，供其他用户查看） |
| crowd_report_id | INTEGER | FK → crowd_reports.id, NULL | 关联本次水文上报表（打卡后可补充上报水文） |

> **变更方式：** ALTER TABLE 脚本添加新列（向后兼容，现有数据 crowd_report_id 为 NULL）
> **注意：** SQLite 外键约束需依赖数据库初始化时设置 `PRAGMA foreign_keys=ON` 才真正生效，请确认 `database.py` 中已配置。

```sql
-- 扩展 spot_checkins
ALTER TABLE spot_checkins ADD COLUMN fishing_method TEXT;
ALTER TABLE spot_checkins ADD COLUMN is_public INTEGER DEFAULT 0;
ALTER TABLE spot_checkins ADD COLUMN crowd_report_id INTEGER REFERENCES crowd_reports(id);
```

## 3. API 接口设计

### 3.1 统一响应格式

所有接口返回格式：
```json
{
  "code": 0,
  "data": { ... },
  "msg": ""
}
```

| code | 说明 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 |
| 403 | 无权限（如修改他人配方） |
| 404 | 资源不存在 |
| 429 | 防垃圾拦截（频率超限或数据越界） |

### 3.6 防垃圾规则详细说明

水文上报 `POST /v1/crowd-reports` 后端拦截逻辑：

**Step 1：关联打卡校验**
- 从 `spot_checkins` 表查出该用户 `spot_id` 下最近一次打卡记录
- 匹配 `checkin_id`（用户传入）是否等于最近打卡 ID，不等则返回 403
- 最近打卡时间距今超过 24 小时，拒绝上报（防止关联陈旧打卡）

**Step 2：数据范围校验**
- `water_temp` 超出 [-5, 40] → 返回 400
- `dissolved_oxygen` 超出 [0, 20] → 返回 400
- `fish_species` 不在标准鱼种列表内 → 返回 400

**Step 3：频率限制校验**
- 查询条件：同 user_id + spot_id + 同一自然日（date(report_time) = today）
- 若已有 N 条记录（默认 N=3，可配置），拒绝本次上报，返回 429

**Step 4：写入 crowd_reports 并更新 checkin.crowd_report_id**

## 5. 关键实现细节

### 5.1 防垃圾机制技术实现

**配置文件（`app/config.py` 或 `app/settings.py`）：**

```python
class ReportSettings:
    WATER_TEMP_MIN: float = -5.0      # 水温下限（°C）
    WATER_TEMP_MAX: float = 40.0      # 水温上限（°C）
    DO_MIN: float = 0.0               # 溶氧量下限（mg/L）
```

**服务层（`app/services/anti_spam_service.py`）：**

```python
class AntiSpamService:
    async def validate_report(self, db, user_id, spot_id, checkin_id, water_temp, dissolved_oxygen, fish_species) -> None:
        # 1. 关联打卡校验
        latest = await self._get_latest_checkin(db, user_id, spot_id)
        if not latest or latest.id != checkin_id:
            raise HTTPException(403, "上报需关联最近一次打卡记录")
        if (now - latest.checkin_time) > timedelta(hours=CHECKIN_WINDOW_HOURS):
            raise HTTPException(403, "打卡已超过24小时，无法上报水文数据")

        # 2. 数据范围校验
        if not (WATER_TEMP_MIN <= water_temp <= WATER_TEMP_MAX):
            raise HTTPException(400, f"水温需在{WATER_TEMP_MIN}~{WATER_TEMP_MAX}°C之间")
        if not (DO_MIN <= dissolved_oxygen <= DO_MAX):
            raise HTTPException(400, f"溶氧量需在{DO_MIN}~{DO_MAX}mg/L之间")
        if fish_species not in FISH_SPECIES_LIST:
            raise HTTPException(400, "鱼种不在标准列表中")

        # 3. 频率限制
        today_count = await self._get_today_report_count(db, user_id, spot_id)
        if today_count >= MAX_REPORTS_PER_DAY:
            raise HTTPException(429, f"今日上报次数已达上限{MAX_REPORTS_PER_DAY}次")
```

```python
async def upsert_rating(db, user_id, spot_id, rating=None, liked=None):
    existing = await db.execute(
        select(SpotRating).where(SpotRating.user_id == user_id, SpotRating.spot_id == spot_id)
    )
    record = existing.scalar_one_or_none()
    if record:
        if rating is not None:
            record.rating = rating
        if liked is not None:
            record.liked = liked
        record.updated_at = datetime.utcnow()
    else:
        record = SpotRating(
            user_id=user_id, spot_id=spot_id,
            rating=rating, liked=liked if liked is not None else True,
            updated_at=datetime.utcnow()
        )
        db.add(record)
    await db.commit()
```
