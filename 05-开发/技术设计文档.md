# 钓了吗 P0 MVP 技术设计文档

## 1. 总体架构

### 1.1 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                     微信小程序（前端）                        │
│  地图页 │ 标点列表 │ 标点详情 │ 天气卡 │ 分享页 │ 设置页     │
└────────────────────────┬────────────────────────────────────┘
                         │ HTTPS (JSON)
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                   Python FastAPI（后端）                      │
│  标点模块 │ 分类模块 │ 禁钓区模块 │ 天气模块 │ 用户模块      │
└────────────────────────┬────────────────────────────────────┘
                         │ SQL
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                      SQLite（本地数据库）                      │
│  fishing_spots / categories / forbidden_zones / weather_cache│
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心设计决策

| 决策项 | 方案 | 理由 |
|--------|------|------|
| 前端形态 | 微信小程序（原生框架） | 微信生态，GPS/分享/支付天然支持 |
| 后端形态 | FastAPI 单体服务 | 轻量，FastAPI 自动生成 OpenAPI，SQLite 免运维 |
| 数据库 | SQLite | 100 用户种子期，数据量小，单文件方便备份 |
| 地图 | 微信地图 API（map 组件） | 小程序原生支持，卫星/路网图层切换 |
| 天气 | 和风天气 API | 免费额度够用，API 稳定，支持实时+逐小时 |
| 标点存储 | 用户私有数据存后端 SQLite | 便于多设备同步，支持后续迁移 |
| 禁钓区 | 后端统一管理，按坐标范围下发 | 避免客户端硬编码，支持动态更新 |
| 微信分享 | URL Scheme（钓了吗://）+ 分享卡片 | 实现简单，支持从微信外拉起小程序 |

### 1.3 数据流

#### 标点 CRUD 流程
```
用户操作 → 小程序 JS → wx.request() → FastAPI → SQLite → 返回 JSON → 小程序 setData() → 地图刷新
```

#### 天气获取流程
```
进入地图页 → 检查缓存（5分钟TTL）→ 有则用缓存 → 无则请求和风API → 写入缓存 → 返回前端
```

#### 禁钓区预警流程
```
小程序 onLocationChange → 计算最近禁钓区距离 → <200m 则弹窗警告 → 用户确认后继续
```

#### 微信分享流程
```
点击分享 → 生成分享卡片（含标点名称+缩略图）→ 分享给好友 → 好友打开 → URL Scheme 解析 → 跳转标点详情页
```

## 5. 安全考虑

- 用户位置数据仅用于禁钓区判断，不持久化
- SQLite 数据库文件存放在服务器非公开目录
- 微信请求需验证 `openid`（简化版不做登录注册，用户即 openid）
- 敏感操作（删除标点）需确认归属

## 6. 可扩展性预留

- 标点表预留 `extra` JSON 字段，支撑未来扩展鱼种/钓法/收费等属性
- 用户表预留 `role` 字段，未来可扩展「管理员发布禁钓区」
- 天气缓存表预留 `raw_response` 字段，方便后续扩展解析字段
