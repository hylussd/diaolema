# 钓了吗 — 分阶段开发规划

> 基于约束条件制定：内陆淡水 / 老钓手 / Python FastAPI / 100用户量级

---

## 约束确认

| 维度 | 决策 |
|---|---|
| 数据来源优先级 | 内陆淡水（水库/野河/湖泊） |
| MVP目标用户 | 老钓手（经验优先，不需要新手引导） |
| 技术栈偏好 | Python快速迭代优先 |
| 预计用户量级 | 100人（种子期） |
| 开发人力 | OpenClaw |

---

## 风险提示（前置）

1. **水文数据缺口** — 内陆淡水无官方API，水温/溶氧量/流速依赖用户众包上报，MVP阶段先接入天气，水文数据用用户上报机制补充
2. **AI推荐质量** — 无真实鱼获数据前，推荐基于规则引擎，不宜夸大效果
3. **社区冷启动** — 需设计众包激励机制补充数据密度

---

## 技术选型

| 层 | 选型 |
|---|---|
| 前端 | 微信小程序（WXML/WXSS/JS） |
| 后端 | Python FastAPI |
| 数据库 | SQLite（早期）→ MySQL |
| 地图 | 微信地图API |
| 天气 | 和风天气API |
| 存储 | 本地文件存储 |

---

## 阶段一：MVP（核心闭环，先跑通）

**目标：** 验证"地图+标点+天气+禁钓区"基础工具价值

### 功能清单

| 功能 | 说明 |
|---|---|
| 地图展示 | 卫星图层 + 路网切换，精准定位 |
| 个人标点CRUD | 上传/编辑/删除标点，补充地形/鱼种/备注 |
| 标点收藏分类 | 创建分类文件夹，批量管理 |
| 基础天气展示 | 温度/气压/风力/湿度，和风API接入 |
| 禁钓区标注 | 红色半透明多边形标注，显示禁钓原因/时间 |
| 禁钓区预警 | 用户进入禁钓区时弹窗警告 |
| 微信分享 | 生成链接，可分享给微信好友 |

### 交付物

- 可运行的微信小程序MVP
- 标点数据结构文档
- 天气API接入文档

---

## 阶段二：差异化（建立竞争壁垒）

**目标：** 让"科学选点"真正落地，与竞品拉开距离

### 功能清单

| 功能 | 说明 |
|---|---|
| 多参数筛选 | 气压区间/水温/目标鱼种/钓场类型组合筛选 |
| AI推荐（简化版） | 基于规则引擎（非模型），输出推荐理由 |
| 标点详情页 | 关联当前天气+历史打卡记录 |
| 微信定向分享 | 生成加密临时链接（有效期可设置） |
| 月相/日出日落 | 天文数据基础展示 |

### 交付物

- 完整标点详情页
- 规则引擎设计文档
- 多参数筛选交互

---

## 阶段三：社区铺垫（数据众包机制）

**目标：** 建立水文数据众包机制，为AI推荐积累真实数据

### 功能清单

| 功能 | 说明 |
|---|---|
| 用户众包上报 | 水温/鱼种/溶氧量上报（需验证机制防垃圾数据） |
| 个人配方保存 | 自定义参数组合保存/命名/分享 |
| 钓点评分/点赞 | 社区氛围基础指标 |
| 历史鱼获记录 | 打卡时间+鱼种+天气+钓法 |

### 交付物

- 众包数据质量控制方案
- 用户配方管理功能
- 鱼获记录数据库设计

---

## 阶段四：商业化（中长期）

**目标：** 建立商业闭环

### 功能清单

| 功能 | 说明 |
|---|---|
| 场景化商城 | 装备推荐/分类/交易（微信支付） |
| 付费会员体系 | 高级AI推荐/优质标点库/免广告 |
| 积分体系 | 社区互动/标点分享积分，兑换优惠券 |

### 交付物

- 商城小程序页面
- 会员体系设计
- 积分商城逻辑

## 开发优先级速查

```
P0 MVP：地图 → 标点CRUD → 基础天气 → 禁钓区 → 微信分享
P1 差异化：多参数筛选 → AI规则推荐 → 标点详情 → 定向分享
P2 社区铺垫：众包上报 → 配方保存 → 点赞评分 → 历史鱼获
P3 商业化：商城 → 会员 → 积分体系
```

---

## 架构说明（MVP阶段）

```
微信小程序
    ↓ HTTP
FastAPI (Python)
    ↓
SQLite（用户数据/标点数据/天气缓存）
    ↓
和风天气API（实时天气）
微信地图API（定位/地图）
```

> 单体服务即可支撑100用户量，无需微服务拆分
